# DEPLOYMENT GUIDE - LAUNCH AT 9:30

## GITHUB PAGES DEPLOYMENT (5 MINUTES)

### STEP 1: CREATE REPOSITORY (2 min)

1. Go to https://github.com
2. Click "+" → "New repository"
3. Repository name: `protocol-seven`
4. Description: "Protocol Seven: Ascension Engine - Track your mission, build your empire"
5. **Public** (required for GitHub Pages)
6. **DO NOT** check "Add README"
7. Click "Create repository"

### STEP 2: UPLOAD FILES (2 min)

1. On your new empty repo, click "uploading an existing file"
2. Drag and drop these 3 REQUIRED files:
   - index.html
   - manifest.json  
   - service-worker.js
3. OPTIONAL files (for documentation):
   - README.md
   - LICENSE
   - DEPLOY.md
4. Commit message: "Initial launch - Protocol Seven v1.0"
5. Click "Commit changes"

### STEP 3: ENABLE GITHUB PAGES (1 min)

1. Click "Settings" tab
2. Click "Pages" in left sidebar
3. Under "Source":
   - Branch: `main` (or `master`)
   - Folder: `/ (root)`
4. Click "Save"
5. Wait 2-3 minutes
6. Refresh the page
7. See: "Your site is live at https://[USERNAME].github.io/protocol-seven/"

## TROUBLESHOOTING

### "Page not found" error:
- Wait 5 full minutes (Pages takes time)
- Check Settings → Pages shows green checkmark
- Verify all 3 files uploaded correctly
- Clear browser cache (Ctrl+Shift+Del)

### Buttons not working:
- Check browser console (F12) for errors
- Verify index.html uploaded completely
- Try different browser

### Share button doesn't work:
- MUST be on HTTPS (GitHub Pages is HTTPS ✓)
- Won't work on file:// protocol
- Test on actual live URL, not locally
